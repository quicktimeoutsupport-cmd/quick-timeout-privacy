Quick Timeout 개인정보처리방침

최종 업데이트: 2026년 7월 15일

Quick Timeout은 개인정보를 수집·추적·판매·공유하지 않습니다. 광고, 분석 도구, 외부 서버 및 인터넷 연결을 사용하지 않습니다.

사용 권한

  • 시스템 설정 변경: 사용자가 프리셋을 선택했을 때만 화면 꺼짐 시간을 변경합니다.
  • 알람 및 리마인더: 선택한 모드가 끝나면 15초 기본값으로 자동 복원되도록 예약합니다.
  • 부팅 후 시작: 기기를 다시 시작한 뒤 대기 중인 자동 복원을 다시 예약합니다.

기기에 저장되는 정보

대기 중인 복원 시각과 화면 꺼짐 시간만 Android 앱 저장공간에 저장될 수 있습니다. 이 정보는 외부로 전송되거나 제3자와 공유되지 않습니다. 복원이 완료 또는 취소되거나, 앱 데이터가 삭제되거나, 앱이 제거되면 삭제됩니다. 개발자는 사용자 데이터를 보관하지 않습니다.

보안

로컬 설정은 Android 앱 샌드박스 안에 저장되며 개발자나 제3자가 접근할 수 없습니다.

방침 변경

이 방침이 변경되면 최종 업데이트 날짜와 함께 이 페이지에 게시합니다.

Privacy Policy for Quick Timeout

Last updated: July 15, 2026

Quick Timeout does not collect, track, sell, or share personal data. It uses no advertising, analytics, external servers, or internet connection.

Permissions

  • Modify system settings: changes the screen timeout only when the user selects a preset.
  • Alarms & reminders: schedules an automatic return to the 15-second default when a selected mode ends.
  • Start after reboot: reschedules a pending automatic restore after the device restarts.

Information stored on the device

Only a pending restore time and timeout value may be stored in Android's app storage. This information is never transmitted or shared with third parties. It is deleted when the restore completes or is canceled, and when app data is cleared or the app is uninstalled. The developer does not retain user data.

Security

Local settings remain in Android's app sandbox and are not available to the developer or third parties.

Changes to this policy

Changes will be posted on this page with a revised last-updated date.
